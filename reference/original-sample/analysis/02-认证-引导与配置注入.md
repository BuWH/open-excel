# 认证、引导与配置注入

## 1. 一号通道：Claude OAuth

本地样本里能直接确认的 OAuth 参数：

- clientId：`966eba67-8b8c-4eae-bbb3-08361d1b9292`
- redirect：`${window.location.origin}/auth/callback`
- scope：
  - `user:profile`
  - `user:inference`
  - `user:file_upload`
  - `org:profile`
  - `user:mcp_servers`
  - `user:voice`

这说明 Excel add-in 不是只拿一个“最小登录 token”，而是直接申请推理、上传、MCP、组织信息等较大权限集合。

## 2. `/auth/callback` 的真实作用

这个回调页不是传统 SPA 内 silent redirect。

它更像一个“把 `code#state` 复制回 add-in” 的中转页：

- 支持复制登录结果
- 在特定平台上可尝试 deep-link 回 Office
- 手工粘贴登录结果的备用流程也在 bundle 里

这很符合 Office add-in 场景：浏览器窗口和任务窗格是两套上下文，不能总依赖标准单页 OAuth 回跳。

## 3. 二号通道：第三方推理入口

产品内建了三类第三方接入：

- Gateway
- Vertex
- Bedrock

对应路由分别是：

- `/auth/3p/gateway`
- `/auth/3p/vertex`
- `/auth/3p/bedrock`

前端页面本身就能收集这些参数，并继续完成接入，不像纯测试遗留。

## 4. manifest / bootstrap 可注入参数

当前样本已识别的 manifest 参数：

- `allow_1p`
- `auto_connect`
- `aws_region`
- `aws_role_arn`
- `bootstrap_url`
- `entra_sso`
- `gateway_auth_header`
- `gateway_token`
- `gateway_url`
- `gcp_project_id`
- `gcp_region`
- `google_client_id`
- `google_client_secret`
- `mcp_servers`
- `otlp_endpoint`
- `otlp_headers`

其中最关键的是四类：

1. 第三方推理参数
   - Gateway / Vertex / Bedrock
2. bootstrap 参数
   - `bootstrap_url`
   - `entra_sso`
3. MCP 参数
   - `mcp_servers`
4. 遥测参数
   - `otlp_endpoint`
   - `otlp_headers`

## 5. bootstrap 解析顺序

`$U(...)` 相关逻辑显示，前端会把配置来源合并起来：

1. manifest query params
2. Entra / NAA claims
3. 远端 bootstrap JSON

然后再合并进前端 store。

观察到的行为包括：

- 触发 `bootstrap_resolved`
- hydrate bootstrap skills
- 决定 provider 默认值
- 决定是否自动跳转到 3P auth 页面

这意味着 bootstrap 不是简单 metadata，而是启动分流逻辑的一部分。

## 6. provider 模型

profile store 里至少支持：

- `oauth`
- `apiKey`
- `gateway`
- `vertex`
- `bedrock`

所以前端内部已经抽象成统一 provider 层，而不只是“Claude 官方登录 + 特殊 case”。

## 7. 认证后的持久化行为

从样本可见：

- profile 会持久化
- `customerConfig` 会持久化
- token refresh / bootstrap refresh 有定时调度
- onboarding / terms 结果走 `localStorage`

相关本地键值包括：

- `claude_excel_terms_accepted`
- `claude_excel_onboarding_completed`

## 8. 结论

这套 add-in 的认证体系不是轻量 login page，而是一个企业可配置接入壳：

- 既能走 Anthropic 一方账号体系
- 也能走企业自己的网关与云模型
- 还能通过 bootstrap 和 claims 动态下发连接参数

从产品定位看，这更接近“Office 里的企业 agent 容器”，而不是单一 SaaS 插件。
