# 总览与结论

## 一句话判断

这个样本本质上是一个统一的 `office-agent` 前端壳层：它在 Office taskpane 中挂载 React 应用，通过 Office.js、Claude OAuth、第三方推理提供商、MCP 连接器和 conductor 跨应用总线，把 Excel/Word/PowerPoint 都接进同一套 agent runtime。

## 核心结论

### 1. 它是统一 Office 前端，而不是“Excel 专属页面”

- 路由、文案、工具描述同时出现 `sheet`、`slide`、`doc` 三种 surface。
- 系统提示、工具定义、能力探测里同时覆盖 Excel、PowerPoint、Word。
- 入口标题虽然是 “Claude in Microsoft Office”，但真实架构更接近 “Claude for Office agent shell”。

### 2. 前端自己承担了不少“编排层”职责

- 并不是单纯把 prompt 发给后端。
- 前端内部负责：
  - 认证与 token 刷新
  - bootstrap 配置解析
  - MCP server 自动接入
  - conductor 连接与共享文件系统
  - Office structured tools 的输入校验、行为约束和安全拦截
  - telemetry / analytics 埋点

### 3. 第三方推理入口是正式能力，不是测试残留

- 入口路由显式存在：
  - `/auth/3p`
  - `/auth/3p/gateway`
  - `/auth/3p/vertex`
  - `/auth/3p/bedrock`
- manifest / claims / bootstrap 可注入对应参数。
- 说明该产品支持企业自带推理网关或云模型后端。

### 4. MCP 与 conductor 是平台级能力

- MCP：
  - 可从 `https://api.anthropic.com/v1/mcp_servers` 拉取远程服务器列表。
  - 也可读取 manifest / bootstrap 中的 `mcp_servers` JSON。
- conductor：
  - 使用 `wss://bridge.claudeusercontent.com`
  - 有 `get_connected_agents`、`send_message`、共享 `/agents/<id>/files/` 文件系统
  - 明确支持 Excel 向 PowerPoint 发送 chart XML

这意味着“Excel 内 Claude”不是孤立 agent，而是一个跨应用协作节点。

### 5. `execute_office_js` 是最关键的高权限工具

- 它把 Office.js 暴露给模型执行代码。
- 同时又用 SES lockdown、代理包装、黑名单函数、OOXML 审查来限制越界能力。
- 这说明产品设计思路不是完全依赖后端工具，而是把一部分宿主操作权下放给前端运行时。

## 当前最稳的逆向判断

1. `pivot.claude.ai` 的 add-in 前端是多 surface 共用的一套 React 工程。
2. `office-agent` 是这套前端 runtime 的统一产品名/服务名。
3. 企业接入路径至少有四种：Claude OAuth、Gateway、Vertex、Bedrock。
4. 连接器能力来自两条线：Anthropic MCP registry + 客户自定义 MCP server。
5. 跨应用协作并非仅消息传递，而是带共享文件、共享 transcript、共享 chart OOXML 的轻量总线。
